# 零基础小程序 Vibe Coding 入门开发助手

一个给零基础用户准备的微信小程序 Vibe Coding 工作流 Skill。

它的目标很简单：

> 让 AI 帮你从一个模糊想法出发，小步、安全、可控地把微信小程序做出来。

这个 Skill 适合想做微信小程序、但不太会写代码的人。  
不管你是产品经理、运营、内容创作者，还是只是突然有个小想法，它都能帮你把一个模糊需求拆成可以落地的小程序开发流程。

---

## 这个 Skill 能帮你做什么？

它可以帮你：

- 把一个想法梳理成小程序 MVP；
- 规划首页、表单页、结果页、个人中心等页面结构；
- 拆解功能优先级，避免一上来就做复杂系统；
- 用 AI 小步写代码、改 UI、修 bug；
- 建立数据结构和本地存储逻辑；
- 检查分享、隐私权限、云开发、发布前配置；
- 生成上线前 checklist；
- 根据真实用户反馈继续迭代。

---

## 核心特点：先给方案，再动代码

这个 Skill 不是让 AI 一上来就乱改代码，而是默认采用：

```text
先给方案 → 你确认 → 再执行 → 做检查 → 总结结果
```

也就是说，涉及这些高风险操作时：

- 改代码；
- 改样式；
- 改配置；
- 改数据；
- 删除文件；
- 调整目录结构；
- 修改云函数；
- 上线发布相关操作；

它会先告诉你：

- 准备改哪里；
- 为什么这么改；
- 哪些地方不动；
- 可能有什么风险；
- 怎么验收。

你确认后，它才会开始执行。

这样可以最大程度避免：

> 功能是做出来了，但项目也被改乱了。

## 推荐使用方式

你只需要用自然语言说清楚：

```text
我想做什么？
给谁用？
最核心的功能是什么？
有没有参考截图或类似产品？
哪些地方暂时不要做？
```

比如：

```text
我想做一个给朋友聚会用的 AA 记账小程序，
主要功能是创建饭局、录入每个人花了多少钱、自动算谁该补给谁。
先不要做登录和复杂报表，只想先跑通核心流程。
```

Skill 会先帮你拆解：

```text
首页 → 创建饭局 → 添加成员 → 录入消费 → 自动结算 → 分享结果
```

然后再给出开发计划，而不是直接一上来改代码。

## 它遵循的工作流

### 1. 明确目标

先搞清楚：

- 用户是谁；
- 解决什么问题；
- 最高频的使用场景是什么；
- 第一版必须做什么；
- 哪些功能先不做。

### 2. 定义 MVP

把想法压缩成最小可用版本，避免一上来就做复杂系统。

例如：

```text
先跑通：填写信息 → 计算结果 → 展示结果
暂不做：账号系统、复杂后台、会员体系、商业化
```

### 3. 先出方案

所有会影响项目结构和代码的操作，都先给方案。

包括：

- 改哪些文件；
- 不改哪些地方；
- 为什么这么做；
- 风险是什么；
- 怎么验收。

### 4. 小步实现

确认方案后，再逐步实现：

```text
页面结构 → 数据结构 → 核心逻辑 → UI 优化 → 异常处理 → 发布检查
```

每一步都尽量小，方便回滚和验收。

### 5. 做检查

改完后会尽量做：

- 语法检查；
- 数据检查；
- 核心路径测试；
- 分享、隐私、云函数检查；
- 上线前 checklist。

### 6. 根据反馈继续迭代

上线后，不急着大重构，而是根据真实用户反馈判断下一步优先级。

## 它会特别注意哪些坑？

做微信小程序时，有很多坑不是新手能提前想到的。

这个 Skill 会特别提醒你：

### 分享能力不是自动有的

小程序备案、审核通过，不代表每个页面都能自动转发。  
需要在页面里配置：

```js
onShareAppMessage()
onShareTimeline()
```

### 朋友圈不能用按钮直接拉起

微信限制页面按钮不能直接打开朋友圈发布面板，只能引导用户从右上角菜单分享。

### 隐私权限能不加就不加

比如保存图片到相册、获取手机号、定位等能力，都会增加审核和隐私说明成本。  
第一版能不用就先不用。

### 状态管理不能乱写

不要在各个页面随手写全局状态，否则后面很容易出现：

```text
上一局的数据残留到下一局
返回页面后状态错乱
分享打开页面数据丢失
```

### 上线前要检查上传包

本地脚本、README、调试文件不要误打进小程序上传包。

## 不适合什么情况？

这个 Skill 不适合一上来就做特别复杂的系统，比如：

- 大型电商平台；
- 完整社交网络；
- 复杂多人实时协作；
- 高并发交易系统；
- 重度后台管理系统；
- 需要强安全合规的金融/医疗类应用。

这类项目需要更完整的技术架构和专业开发流程。
